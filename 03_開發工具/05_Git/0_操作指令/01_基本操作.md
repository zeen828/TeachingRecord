## **Teaching & Examples [教學&範例]**

### 初始化
```bash
# 初始話專案
git init
# 設定用戶資訊
git config --global user.email "user@test.com"
git config --global user.name "user name"
# 查看遠端來源
git remote -v
```

### 拉取專案
```bash
# []是選填，沒另外命名目錄預設會是TeachingRecord
git clone https://github.com/zeen828/TeachingRecord.git [另外命名目錄]
```

### 查詢狀態
```bash
# 查看 Git 狀態
git status
# 查看 Git 歷史
git log
git log --oneline --graph
```

### 分支
```bash
# 查詢本地分支
git branch
# 查詢線上分支
git branch -r
# 刪除本地分支
git branch -d <branch_name> <branch_name>…
# 刪除本地分支(強制)
git branch -D <branch_name> <branch_name>…
# 切換本地分支
git checkout <branch_name>
git switch <branch_name>
# 建立並切換分支
git checkout -b <branch_name>
git switch -c <branch_name>
```

### 拉取
```bash
# 抓取更新(只下載不合併)
git fetch
# 合併
git merge
# 拉取並合併(等同上面兩步驟)
git pull
git pull origin master
```

### 提交
```bash
# 新增檔案
git add file.php
# 新增全部異動檔案
git add .
# 提交 Commit
git commit -m "新增登入功能"
# 修改最後一次 Commit 訊息
git commit --amend -m "修正訊息"
# 推送到線上(master線上分支名稱)
git push origin master
```

### 查看異常
```bash
git diff
git diff file.php
git diff --staged
```

### 暫存
```bash
# 儲存目前修改
git stash
# 查看
git stash list
# 還原
git stash pop
# 還原但保留
git stash apply
```

### 標籤
```bash
# 建立版本
git tag v1.0.0
# 推送
git push origin v1.0.0
```

### 退版(Commit)
```bash
# Commit 時發現有漏檔案或訊息錯誤可以用這個返回處理
# 退一個 Commit 並保留異動檔案且異動是 add 中
git reset --soft HEAD~1

# Commit 時發現有提交錯誤的檔案可以用這個返回處理
# 退一個 Commit 並保留異動檔案非 add 中(同 git reset HEAD~1)
git reset --mixed HEAD~1
git reset HEAD~1

# Commit 時發現有提交錯誤的全部捨棄
# 退一個 Commit 並刪除所有異動檔案
git reset --hard HEAD~1
```


假設 C 有修改：
|指令	|Commit	|Stage	|檔案|
|-------|-------|------|----|
|soft	|刪除	|保留	|保留|
|mixed	|刪除	|清除	|保留|
|hard	|刪除	|清除	|刪除|


### 退版
```bash
git revert <commit>

git revert -m 1 <merge_commit>
```

### 整理分支
```bash
git rebase master
# 整理 Commit
git rebase -i HEAD~5
```
